## 📐 异常状态统一标准 / Unified Status Standards

> **重要**: 所有报告类型必须遵循 `references/status_standards.md` 中定义的统一标准

### 状态图标速查 / Status Icons Quick Reference

| 异常级别 | 图标 | 显示格式 | 适用范围 |
|----------|------|----------|----------|
| **0级 正常** | (无标记) | `数值` | 所有报告 |
| **I级 轻度** | ⚠️ | `⚠️数值↓` / `⚠️数值↑` | 血常规/血脂 |
| **II级 中度** | 🟠 | `🟠数值↓↓` / `🟠数值↑↑` | 血常规/血脂 |
| **III级 重度** | 🔴 | `🔴数值↓↓↓` / `🔴数值↑↑↑` | 所有报告 |
| **IV级 危急** | 🚨 | `🚨数值↓↓↓↓` | 血常规危急值 |

> **核心规则**: 正常指标不加任何图标或✅，仅显示数值！异常时才添加图标+方向箭头！

### 趋势箭头规范 / Trend Arrows

| 箭头 | 含义 | 使用场景 |
|------|------|----------|
| ↑ | 高于正常上限 | 数值异常偏高时 |
| ↓ | 低于正常下限 | 数值异常偏低时 |
| ↗ | 较上次升高 | 与历史对比 |
| ↘ | 较上次降低 | 与历史对比 |

### CT影像专用状态 / Imaging-Specific Status

| 评价 | 图标 | 关键词示例 |
|------|------|-----------|
| 稳定/好转 | ✅ | 稳定、好转、缩小、吸收 |
| 稳定但关注 | ⚠️ | 稳定但有炎症增多、新发病变 |
| 进展/恶化 | 🔴 | 进展、增大、转移 |

**完整标准文档**: [references/status_standards.md](./references/status_standards.md)

## ⚠️ 危急值警示 / Emergency Warning Signs（最高优先级）

**出现以下情况需立即就医 / Immediate medical attention required if:**

| 危急值 | 风险描述 |
|--------|----------|
| ANC < 0.5×10⁹/L | 重度中性粒细胞减少，高感染风险 |
| PLT < 20×10⁹/L | 出血危象，自发性出血风险 |
| Hb < 60 g/L | 重度贫血，心脏并发症风险 |
| 体温 ≥ 38.0℃ + 低ANC | 发热性中性粒细胞减少症，医疗急症 |
| 活动性出血 | 牙龈出血、鼻出血、便血/血尿 |

> **处理原则**: 发现上述情况立即联系主治医生或前往急诊！

## 支持的报告类型 / Supported Report Types

### 1. 血常规 (Blood Routine)

**核心指标 / Core Indicators：**

| 指标 | 中文名称 | 正常范围(成人) | 单位 |
|------|----------|---------------|------|
| WBC | 白细胞 | ≥4.0 | ×10⁹/L |
| ANC | 中性粒细胞 | ≥1.5 | ×10⁹/L |
| PLT | 血小板 | ≥100 | ×10⁹/L |
| Hb | 血红蛋白 | 男≥120, 女≥110 | g/L |
| CRP | C反应蛋白 | ≤10 | mg/L |
| LYM# | 淋巴细胞计数 | ≥1.1 | ×10⁰/L |

### 2. 血脂 (Lipid Profile)

**关注指标：**
- 总胆固醇 (TC)
- 甘油三酯 (TG)  
- 高密度脂蛋白 (HDL-C)
- 低密度脂蛋白 (LDL-C)

### 3. 肿瘤标志物 (Tumor Markers)

**关注指标 / Key Indicators：**

| 指标 | 中文名称 | 正常范围 | 单位 |
|------|----------|---------|------|
| CA-125 | 糖类抗原125 | 0－35 | U/mL |
| CYFRA21-1 | 细胞角蛋白19片段 | 0-3.3 | ng/mL |
| VEGF | 血管内皮生长因子 | 0-160 | pg/mL |
| SCC | 鳞状细胞癌相关抗原 | 0-3 | ng/ml |

**状态标识：**
- ✅ 正常范围内 → 治疗有效，继续监测
- 🟠 轻度升高 → 关注趋势，定期复查
- 🔴 明显升高 → 需关注病情进展

### 4. CT/MRI影像报告 (Imaging Reports)

**记录内容：**
- 检查日期、医院
- 原发灶情况
- 转移灶变化
- 总体评价
- 医生建议

## 核心工作流 / Core Workflow

### 第一步：接收并识别报告图像 / Receive & Recognize Report

当用户上传医疗报告图片时：

1. **识别报告类型**
   - 通过标题和指标名称判断是哪种报告
   - 血常规、血脂、肿瘤标志物、CT/MRI等

2. **提取关键信息**
   - 检查日期
   - 医院/实验室名称
   - 所有检测指标及数值
   - 参考范围（如有显示）

3. **数据提取最佳实践**
   - 使用视觉能力准确读取数字
   - 注意单位（WBC/ANC/PLT通常为×10⁹/L，Hb为g/L）
   - 图像不清晰时明确说明，不猜测或编造数据

### 第二步：基于参考文档评估异常 / Evaluate Using Reference Document

使用 `references/blood_test_reference.json` 作为**唯一权威依据**判断异常。

**分级标准 / Grading System：**

| 级别 | 含义 | 处理优先级 |
|------|------|-----------|
| 0级 | 正常 | 无需处理 |
| I级 | 轻度异常 | 观察监测 |
| II级 | 中度异常 | 建议就诊 |
| III级 | 重度异常 | 紧急就医 |
| IV级 | 极重度 | 立即就医 |

**特殊注意事项：**
- **Hb（血红蛋白）**：需根据性别判断 - 男性≥120g/L，女性≥110g/L
- **ANC（中性粒细胞）**：<0.5×10⁹/L为感染高风险阈值
- **PLT（血小板）**：<20×10⁹/L为出血危象阈值

### 第三步：生成综合报告 & 自动录入 / Generate Report & Auto-Record

#### 📊 指标分析表模板 / Indicator Analysis Table

```
| 指标名称 | 检测值 | 参考范围 | 异常级别 | 状态 |
|---------|-------|---------|---------|------|
| [Name]  | [Value]| [Range] | [Level]  | [Status]|
```

**状态标识 / Status Indicators（遵循统一标准）：**
- **0级 正常 Normal** → 仅显示 `数值`（**禁止加✅或任何图标！**）
- ⚠️ I级轻度异常 Mild → `⚠️数值↓` / `⚠️数值↑`
- 🟠 II级中度异常 Moderate → `🟠数值↓↓` / `🟠数值↑↑`
- 🔴 III级重度异常 Severe → `🔴数值↓↓↓` / `🔴数值↑↑↑`
- 🚨 IV级极重度 Critical → `🚨数值↓↓↓↓` (需立即关注)

#### 💡 处理建议 / Handling Recommendations

对每个异常指标提供：
1. **处理原则 Treatment Principles**
2. **出院医嘱 Discharge Orders** - 用药指导
3. **自我监测 Self-Monitoring** - 症状观察要点
4. **生活护理 Life Care** - 饮食活动注意

💡 温馨提示：

4. 如出现不适症状及时就医

### 第四步：更新飞书文档台账 / Update Feishu Spreadsheet

> **⚠️⚠️⚠️ 最高优先级约束 ⚠️⚠️⚠️**
>
> **绝对禁止手动拼装数据直接调用飞书API或让用户复制粘贴到表格！**
>
> **必须且只能调用 `scripts/feishu_api.py` 中的函数来写入数据！**
>
> 违反此规则将导致：时间错乱、格式不一致、列错位等问题。

#### 4.0 唯一正确的写入方式（强制）

**CT报告必须调用 `smart_archive_ct()` 函数：**

```python
# 正确做法 ✅ - 在终端执行：
python3 ~/.codebuddy/skills/medical-record-butler/scripts/feishu_api.py << 'EOF'
# 或者通过 Python 调用 smart_archive_ct():
from scripts.feishu_api import smart_archive_ct

result = smart_archive_ct({
    "date": "2025-03-07",              # 必填: YYYY-MM-DD
    "hospital": "医科院肿瘤医院",       # 必填
    "exam_type": "颈+胸+腹CT平扫",
    "primary_focus": "✅左肺上叶支气管腔",
    "metastasis": "右下肺叶结节0.8;",
    "overall_eval": "✅整体病情稳定",   # 必须 ≤15字符!
    "doctor_advice": "继续定期复查监测"
})
print(result)
EOF
```

**脚本会自动完成以下所有工作（不需要AI手动做）：**
1. ✅ 认证飞书 API
2. ✅ 读取目标工作表的现有数据和表头
3. ✅ 按日期自动计算插入位置（时间倒序）
4. ✅ 校验每列长度和格式（超长则拒绝写入并报错）
5. ✅ 检查重复数据
6. ✅ 写入到正确位置

#### 4.1 当前飞书表格实际结构（CT影像报告工作表）

config.json 已配置的实际列顺序（**必须严格匹配**）：

| 列号 | 列名 | 格式要求 | 最大长度 | 示例 |
|------|------|----------|----------|------|
| A | **检查日期** | YYYY-MM-DD | 12字符 | `2025-03-07` |
| B | **医院** | 与历史一致的简称 | 20字符 | `医科院肿瘤医院` |
| C | **总体评价** | **图标 + ≤8字短语** | **15字符** | `✅整体病情稳定` |
| D | **重点关注** | 关键异常/变化点 | 25字符 | `右肺结节大小` |
| E | **处理建议** | 可操作的建议 | 20字符 | `定期复查监测` |
| F | **检查类型** | CT/MRI部位+方式 | 18字符 | `颈+胸+腹CT平扫` |
| G | **原发灶(左肺)** | 图标+状态描述 | 25字符 | `✅左上叶支气管腔` |
| H | **右肺结节** | 大小+变化 | 25字符 | `结节0.8cm;稳定` |
| I | **淋巴结情况** | 描述 | 40字符 | `多发淋巴结0.6cm` |

> **C列(总体评价)是最容易出错的列！** 超过15字符会被脚本**拒绝写入**。
> 
> ❌ 错误: `✅免疫治疗后部分炎性病变需有`(19字符)
> 
> ✅ 正确: `✅治疗后部分炎变`(10字符)

#### 4.2 各字段内容提取指南

从CT报告中提取数据时：

| 字段 | 从哪里提取 | 怎么提炼 |
|------|-----------|----------|
| **date** | 报告头部的检查日期 | 直接取，格式化为YYYY-MM-DD |
| **hospital** | 报告头部的医疗机构名 | 取简称，与历史保持一致 |
| **exam_type** | 检查方法描述 | 如"胸部CT平扫""颈胸腹增强CT" |
| **primary_focus** | 影像所见中关于左肺/主病灶的描述 | 压缩为图标+一句话 |
| **metastasis** | 关于转移灶/结节的描述 | 提取大小数字+变化趋势 |
| **overall_eval** | 结论/诊断意见的最后一句 | **压缩到8字以内**加图标 |
| **doctor_advice** | 建议随访部分 | 提炼为可操作的动作 |

**总体评价(overall_eval)压缩技巧：**
- "免疫治疗后部分炎性病变需有..." → `⚠️炎变较前增多`
- "整体病情稳定可控，建议继续当前方案" → `✅整体病情稳定`
- "病变较前进展，出现新发转移灶" → `🔴病变有进展`

#### 4.3 错误恢复

如果脚本返回校验错误（如某列过长）：
1. 读取错误信息中指出的具体列和问题
2. 修正该字段内容（缩短/改格式）
3. **重新调用** `smart_archive_ct()` ，不要尝试手动写入

```python
# 校验失败时的典型返回:
{
    "success": False,
    "validation_failed": True,
    "errors": [
        "第3列[总体评价]内容过长(19字符>15上限): '✅免疫治疗后部分炎性...' (要求: 图标+≤8字短语)"
    ],
    "info": "格式校验不通过:\n..."
}
# → 修正 overall_eval 为更短的文本后重新调用即可
```

### 沟通风格 / Communication Style

## 🔑 本地配置说明（重要）

### 飞书集成配置

本 skill 支持将报告数据自动录入飞书电子表格。使用此功能需要配置飞书应用凭证：

#### 步骤1：获取飞书应用凭证

1. 登录 [飞书开放平台](https://open.feishu.cn/)
2. 创建或进入你的自建应用
3. 在「凭证与基础信息」页面获取：
   - **App ID** (格式: `cli_xxxxxxxxx`)
   - **App Secret** (格式: `xxxxxxxx`)

#### 步骤2：创建本地配置文件

在 **skill 同级目录** 创建 `config.json` 文件，内容如下：

```json
{
  "app_id": "cli_你的AppID",
  "app_secret": "你的AppSecret",
  "spreadsheet_token": "你的电子表格Token",
  "sheets": {
    "blood_routine": {
      "name": "血常规",
      "enabled": true,
      "columns": ["日期", "医院", "WBC", "ANC", "PLT", "Hb", "异常", "建议"]
    },
    "ct_report": {
      "name": "CT报告",
      "enabled": true,
      "columns": ["日期", "医院", "检查类型", "原发灶", "转移灶", "总体评价", "医生建议"]
    }
  },
  "settings": {
    "deduplication": { "enabled": true, "key_fields": ["date", "hospital", "report_type"] },
    "emergency_warning": true
  }
}
```

#### 步骤3：获取电子表格 Token

1. 打开或创建一个 [飞书电子表格](https://p16aafnobs.feishu.cn/sheets)
2. 从浏览器地址栏复制 Token: `https://p16aafnobs.feishu.cn/sheets/{TOKEN}`
3. 填入 `spreadsheet_token` 字段

### 配置文件位置

| 环境 | 配置路径 |
|------|----------|
| CodeBuddy Skill | `~/.codebuddy/skills/病历管家/config.json` |
| 或 | skill 目录下的 `config.json` |

> **安全提示**: `config.json` 包含敏感凭证，已加入 `.gitignore`，**不会被推送到 GitHub**

### 不配置飞书？

如果不需要自动归档到飞书表格，skill 仍可正常使用所有识别和分析功能，只是数据会以文本形式输出而非写入表格。
